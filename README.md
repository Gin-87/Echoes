# Echoes

An AI-powered letter companion platform where users exchange letters with fictional characters generated by LLMs. Users configure a character relationship, and the system periodically generates personalized letters in the character's voice — with full conversation history as multi-turn context.

---

## Architecture Overview

```
┌─────────────────┐         ┌─────────────────────┐         ┌──────────────────┐
│   React Frontend │ ──────▶ │  Java Spring Boot   │ ──────▶ │ Python FastAPI   │
│   (Vite)        │         │  (REST API / Auth)  │         │ (LLM Service)    │
│  localhost:5173 │         │   localhost:8080    │         │  localhost:8001  │
└─────────────────┘         └─────────────────────┘         └──────────────────┘
                                       │                               │
                                       ▼                               ▼
                                  ┌─────────┐                  ┌─────────────┐
                                  │  MySQL  │                  │  Doubao/GPT │
                                  │ (Users, │                  │    (LLM)    │
                                  │  Tasks) │                  └─────────────┘
                                  └─────────┘
```

**Communication flow:**

1. Java receives a letter task creation request
2. Java aggregates config (character + task + model) and POSTs to Python (202 Accepted)
3. Python executes LLM call asynchronously via `BackgroundTasks`
4. Python POSTs generated letter back to Java `/api/integration/callback`
5. Java persists the letter to `letter_task_logs`

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/your-username/Echoes.git
cd Echoes
```

### 2. Set up MySQL

```bash
mysql -u root -p
```

```sql
CREATE DATABASE echoes CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
exit;
```

### 3. Configure the Java backend

Edit `src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/echoes?useSSL=false&serverTimezone=UTC
spring.datasource.username=root
spring.datasource.password=YOUR_MYSQL_PASSWORD

python.service.url=http://localhost:8001
```

### 4. Start the Java backend

```bash
mvn spring-boot:run
```

Wait for:

```
Started EchosApplication in X.XXX seconds
```

API docs available at: `http://localhost:8080/swagger-ui/index.html`

### 5. Install Python dependencies

```bash
cd src/main/Python/IntegrationService
pip3 install fastapi uvicorn requests pydantic
```

### 6. Start the Python AI service

```bash
python3 main.py
```

Wait for:

```
Uvicorn running on http://0.0.0.0:8001
```

API docs available at: `http://localhost:8001/docs`

### 7. Start the frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend available at: `http://localhost:5173`

## Configure an LLM Model

After starting the backend, insert your model config directly into MySQL:

**For Doubao (volcengine):**

```sql
INSERT INTO ai_model_configs (model_name, api_address, api_key, code, max_token, created_at, updated_at)
VALUES (
  'your-endpoint-id',               -- e.g. ep-20260404xxxxxx-xxxxx
  'https://ark.cn-beijing.volces.com/api/v3/chat/completions',
  'your-api-key',
  'doubao',
  1000,
  NOW(), NOW()
);
```

**For OpenAI GPT:**

```sql
INSERT INTO ai_model_configs (model_name, api_address, api_key, code, max_token, created_at, updated_at)
VALUES (
  'gpt-4o',
  'https://api.openai.com/v1/chat/completions',
  'sk-your-api-key',
  'gpt',
  1000,
  NOW(), NOW()
);
```

## Register a User

Password must be RSA-encrypted before sending. Use this helper script:

```bash
pip3 install pycryptodome

python3 << 'EOF'
import urllib.request, json, base64
from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_v1_5

with urllib.request.urlopen("http://localhost:8080/rsa-service/rsa/public-key") as r:
    pub = r.read().decode()

cipher = PKCS1_v1_5.new(RSA.import_key(f"-----BEGIN PUBLIC KEY-----\n{pub}\n-----END PUBLIC KEY-----"))
encrypted = base64.b64encode(cipher.encrypt(b"your-password")).decode()

data = json.dumps({
    "email": "you@example.com",
    "password": encrypted,
    "nickname": "your-nickname",
    "role_id": 2,
    "status": "ACTIVE"
}).encode()

req = urllib.request.Request(
    "http://localhost:8080/api/users/register",
    data=data,
    headers={"Content-Type": "application/json"},
    method="POST"
)
with urllib.request.urlopen(req) as r:
    print(json.dumps(json.loads(r.read()), indent=2))
EOF
```

## Key Design Decisions

**Strategy Pattern for LLM adapters**
Adding a new model backend (e.g. Claude) requires only a new `Adapter` class registered in `LLMAdapterRegistry` — zero changes to the core request flow.

**Two-layer async communication**

- `BackgroundTasks`: keeps the HTTP interface non-blocking (202 Accepted), eliminating timeout risks on long LLM calls
- Callback endpoint (`POST /api/integration/callback`): decouples result persistence from LLM execution

**Scheduled letter delivery**
`LetterScheduler` runs daily at 02:00, evaluating each `ENABLED` task against its `FrequencyStatus` enum (`WEEKLY` / `HALFMONTHLY` / `MONTHLY` / `RANDOM`) to decide whether to trigger generation that day.

## Notes

- On every Java service restart, JWT keys are regenerated. Clear `localStorage` in the browser and re-login to get a new token.
- The `menu` table uses `order` as a column name which conflicts with MySQL reserved keywords — this throws a warning on startup but does not affect functionality.
- Default scheduler trigger time is 02:00 local time. Modify the cron expression in `LetterScheduler.java` to change this.
